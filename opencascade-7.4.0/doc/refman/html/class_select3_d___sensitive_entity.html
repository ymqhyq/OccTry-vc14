◆ GetConnected()

virtual Handle< Select3D_SensitiveEntity > Select3D_SensitiveEntity::GetConnected ( )
inlinevirtual

Originally this method intended to return sensitive entity with new location aLocation, but currently sensitive entities do not hold a location, instead HasLocation() and Location() methods call corresponding entity owner's methods. Thus all entities returned by GetConnected() share the same location propagated from corresponding selectable object. You must redefine this function for any type of sensitive entity which can accept another connected sensitive entity.

Reimplemented in Select3D_SensitivePrimitiveArray, Select3D_SensitiveGroup, Select3D_SensitiveCircle, MeshVS_CommonSensitiveEntity, Select3D_SensitiveFace, Select3D_SensitiveTriangulation, Select3D_SensitiveTriangle, Select3D_SensitiveCurve, MeshVS_SensitivePolyhedron, Select3D_SensitiveSegment, Select3D_SensitiveBox, MeshVS_SensitiveQuad, MeshVS_SensitiveMesh, Select3D_SensitiveWire, and Select3D_SensitivePoint.
